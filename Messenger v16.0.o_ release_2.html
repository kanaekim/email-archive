Aurea Messenger 16.0.0: Java 21 Support & Major Platform Updates

Dear Aurea Messenger Customer,

 

We are pleased to announce the release of Aurea Messenger 16.0.0 (2026 R3).
This is a major release introducing Java 21 support, platform modernization, and extensive fixes across broker, ESB, and tooling components.

Platform Changes

  • Java 21 is now mandatory (installer will not run on earlier JVMs)
  • Eclipse workbench upgraded from 4.30 → 4.39
  • Full validation across:
    • Broker
    • ESB container
    • Management framework
    • SDM and workbench

Key Enhancements

Broker & ESB Reliability

  • Improved shutdown sequencing and thread lifecycle handling
  • Hardened fault-tolerant peer reconnection
  • Fixed broker startup hangs due to malformed configurations
  • Corrected ESB routing for sonic:///local process addressing

SSL/TLS Improvements

  • Enhanced diagnostics for unsupported cipher configurations
  • Automatic inclusion of TLS 1.3 ciphers alongside TLS 1.2
  • Improved handling of invalid cipher lists with actionable logging

Jakarta EE Support

  • New Jakarta-compatible MQ client JARs (-jakarta artifacts)
  • Continued support for existing javax.* clients
  • BOM and migration guidance included

Tooling & Workbench

  • Stable handling of .esbp files and editors
  • Fixed XSLT editor rendering issues
  • Improved Project Explorer behavior

Docker Distribution Updates

  • Docker images now run on ARM hosts (e.g., Apple Silicon Macs)ARM-compatible builds (linux/amd64 platform support)
  • Increased Domain Manager memory (2GB)
  • Removal of embedded license keys in samples
  • CX Deployer CLI now bundled

Security Updates

  • 9 CVEs resolved across:
    • Xerces (upgraded to 2.12.2)
    • Bouncy Castle (upgraded to 1.78)

Upgrade Considerations

  • Java 21 required — upgrade JVM before installation
  • Review SSL_CIPHER_SUITES configurations:
    • Remove legacy ciphers (e.g., TLS_ECDH_, TLS_KRB5_, 3DES)
    • Prefer JVM-managed cipher selection where possible
  • Back up Eclipse workspaces prior to automatic migration
  • SDM now prioritizes JAVA_HOME over default.properties

Notable Fixes

  • HTTPS client certificate authentication restored for modern Java runtimes
  • REST payload logging and HTTP Direct routing issues resolved
  • Split-Join fault propagation corrected
  • Delivery limit topic forwarding issues fixed

This release ensures compatibility with modern Java platforms while significantly improving operational stability and security posture.
