Reminder: Aurea Monitor 2026R1 — Upgrade to Java 21 & Enhanced Security

Dear Aurea Monitor Customer,

 

Just a friendly follow-up that Aurea Monitor 2026R1 (Version 16.x) is available and ready for download.

 

As previously announced, this major platform update modernizes the product runtime, strengthens security, and validates all shipping interceptors on current Java technology. The release upgrades the platform from Java 8 to Java 21, introduces Jakarta EE namespace support for modern application servers, resolves numerous known security vulnerabilities, and includes end-to-end validation of every supported interceptor.

 

What’s new in Aurea Monitor 2026R1

  • Java 21 runtime support: Aurea Monitor now runs on Java 21, the current long-term support standard, for the Management Server, Agent, and Intermediary.
  • Jakarta EE support: Interceptors have been updated for jakarta.* namespace used by modern application servers such as Tomcat 10.1+, JBoss EAP 8.1+, and webMethods 11.1+.
  • Interceptor validation on Java 21: Every shipping interceptor has been validated end-to-end to confirm service interactions are captured and displayed correctly in the Monitor dashboard.
  • Security updates: This release addresses multiple security issues through library upgrades, component modernization, and the removal of unsupported legacy modules.

Important upgrade notes

  • A new license key is required: Existing license keys from prior releases will not work with 2026R1. A new v16.0 license key must be obtained before upgrading.
  • Java 21 is required: Java 8 is no longer supported for the Management Server, Agent, or Intermediary. Amazon Corretto 21 is the recommended distribution.
  • Interceptors remain Java 8 compatible: The interceptor JARs deployed to monitored applications continue to use Java 8 bytecode, so monitored application servers do not need to move to Java 21.
  • Jakarta interceptor distribution only: The 2026R1 distribution includes only Jakarta EE interceptors. Customers running older application servers that require javax.* interceptors can continue using 14.x interceptors with the 2026R1 server-side components.

Notable changes in this release

  • Runtime upgraded from Java 8 to Java 21
  • Full validation of all shipping interceptors on Java 21
  • Jakarta EE interceptor support added
  • Legacy and end-of-life interceptors removed
  • Multiple security vulnerabilities remediated
  • Jetty was upgraded as part of the platform modernization
  • Experimental Kafka, Elasticsearch, and Cassandra-related components removed

What this means for you

Aurea Monitor 2026R1 provides a more secure, modern, and supportable platform foundation while preserving flexibility for existing monitored environments. You can upgrade core server-side components to Java 21 while continuing to monitor applications that still run on Java 8, provided you use the appropriate interceptor distribution.


This release is particularly important for customers planning to modernize infrastructure, adopt newer application servers, or reduce exposure to older platform and library vulnerabilities.

 

Next steps

Before upgrading, please make sure you:

  • Obtain a new v16.0 license key
  • Install Java 21 on the Management Server, Agent, and Intermediary hosts
  • Review the installation and upgrade guide
  • Plan interceptor redeployment using the Jakarta EE interceptor distribution, where applicable
